版本：test4
时间：2018.10.17+2018.10.18
作者：李文宽

总述：在test3的基础上添加Eigen库，定义变量和初值。

详情：
2018.10.17
  1、添加Eigen库。
	把eigen库的文件夹“Eigen”添加到“JNI”文件夹下，然后在cpp文件中申明：#include 	<Eigen/Core>
	#include <Eigen/Dense>
	using namespace Eigen;
	同时在cmakelist文件中添加：
	# include_directories("E:/51_AndroidLearn/eigen-eigen-b3f3d4950030/Eigen")
	include_directories( "E:/51_AndroidLearn/0_android-ndk-master/other-builds/ndkbuild/test-libstdc++/jni" )
  2、定义变量和赋初值：
	显示搞明白了全局变量和局部变量，局部变量在函数（如本项目中的init，update等）中定义，且每次使用的时候都会对其重新定义。所以，对于磁场初始值，以及一些过程中不需要重复重新定义的量要设为全局变量。
	同时，由于对android studio中的eigen使用不明了，导致这些花费了很长的时间。
结果：给磁场初始值一个定值（50,0,0），可以通过陀螺仪数据对其进行旋转。

  3、把函数的注释全删除了，如果需要查看函数的定义，可参考https://developer.android.google.cn/ndk/reference/group/sensor

2018.10.18
  1、陀螺仪数据的一个简单滤波。
  	在昨天对一个常矢量进行旋转的过程中发现，陀螺仪数据变化过大，旋转过程中，值会变化很大，所以添加一个见得滤波，对上一时刻的陀螺仪数据取值0.2，此时刻的陀螺仪数据之取值0.8，进行融合。
  2、添加EKF磁场校准算法
	添加了之后，发现磁场校准后的数据不对劲，应该是逻辑上还有错误。
  3、磁场初始值。
	在确定磁场初始值时发现。如果把真正的第一个磁场值给进去，会不稳定。所以打算添加一个“time-begin”在time_begin等于50，即0.5s时，把此时的磁场值作为初始磁场值。发现稳定了一些。

问题：
  0、初始化P矩阵！
  1、最终得到的EKF磁场校准数据还是不太对劲。
	2018.10.18. 17:00 经过磁场初值的更改添加之后，校准后的磁场数据从图上看是没什么问题。
  2、重点！！如何把数据写出到一个文件夹？*********
  3，如果把同一个程序写进两个app？

  1、仅适用陀螺仪数据对初始磁场值进行旋转，发现会漂移。但是对磁场干扰没有任何反应。说明陀螺仪方法可以屏蔽周围环境磁场的干扰。
